## 1. 卷积——计算机视觉的"放大镜"

### 1.1 为什么卷积能"看懂"图像？

**故事开始**：全连接层也能处理图像——把 224×224×3 的图像展平成 150528 个数，输入一个巨大的全连接层。但这样做有两大致命问题：

1. **参数量爆炸**：`150528 × 第一层神经元数`，轻松上亿参数
2. **丢失空间信息**：展平后，像素 (0,0) 和 (0,1) 的距离跟 (0,0) 和 (223,223) 一样远——"邻居"的概念消失了

卷积用两个先验知识解决了这两个问题：
- **局部性**：一个像素只和它周围的像素有关系（"猫的耳朵在猫的眼睛旁边，不在 200 像素之外"）
- **平移等变性**：一个特征在图像左边和右边的识别方式应该一样（"猫耳朵不管在哪都是猫耳朵"）

### 1.2 六个参数——卷积的"控制面板"

```
输入: [B, C_in, H, W]    输出: [B, C_out, H', W']
权重: [C_out, C_in/groups, kH, kW]    ← 这就是"滤波器"

输出尺寸 = ⌊(H + 2×pad − dilation×(k−1) − 1) / stride + 1⌋
```

| 参数 | 作用 | 常见选择 | 一句话总结 |
|------|------|---------|-----------|
| **kernel** | "看多大范围" | 3×3 最经典 | 两个 3×3 = 一个 5×5 的感受野，但参数更少 |
| **stride** | "跳多快" | 1=保持, 2=减半 | stride=2 可以替代 MaxPool 做下采样 |
| **padding** | "边界怎么处理" | same=输出不变 | 不加 padding 的话每层都会缩小 |
| **dilation** | "看多远（空洞）" | 1=紧密, 2+=稀疏 | 不看邻居看远方，适合语义分割 |
| **groups** | "分几组干活" | 1=标准, C_in=深度 | MobileNet 的秘密武器：分组数=通道数 |
| **bias** | "要不要加偏置" | 接BN→关 | BN 已经自带偏置效果，再加就是冗余 |

### 1.3 四种卷积算法——同一道题，四种解法

```
3×3 卷积, stride=1:
  Direct:     逐位置算，简单但慢
  im2col+GEMM: 展开成矩阵列，用 GEMM 算，内存膨胀 9 倍
  Winograd:    变换到另一个空间，点乘替代卷积，快 1.5-2× 但精度略损
  FFT:         频域点乘，大核才划算

ncnn 的选择逻辑:
  3×3 stride=1 → Winograd (最优)
  1×1          → Direct 特殊路径 (等价于 MatMul)
  通用 k×k     → im2col+GEMM (稳定可靠)
  Depthwise    → Direct (数据量小，im2col 不划算)
  dilation>1   → Direct (Winograd 不支持空洞)
```

> 💡 **Winograd 的直觉**：正常做 3×3 卷积每个位置要 9 次乘法。Winograd 通过一个巧妙的变换，把 9 次乘法变成 4 次——代价是多了几次加法（加法比乘法便宜得多）。对于 3×3 这种小卷积核，收益非常显著。

### 1.4 感受野——"这一层能看到多远？"

```
单层: RF = dilation × (kernel−1) + 1

多层累积:
  RF_L = RF_{L-1} + (kernel_L − 1) × (前面所有 stride 的乘积)

例子: 3 个 stride=1 的 3×3 卷积
  RF₁=3, RF₂=3+2×1=5, RF₃=5+2×1=7
  → 第三层的一个像素"看到"了原始图像的 7×7 区域

等价于一个 7×7 卷积，但参数量: 3×9=27 < 49
→ 这就是 VGG 用"小卷积核堆叠"替代"大卷积核"的数学依据
```

## 2. 内存布局——NCHW 还是 NHWC？

这不是一个学术问题——同一个模型，换一种内存排列，在手机上可能快 2 倍。

```
NCHW (PyTorch/ncnn 默认):   [N][C][H][W]
  内存: N₀C₀H₀W₀, N₀C₀H₀W₁, ..., N₀C₀H₁W₀, ..., N₀C₁H₀W₀, ...
  优点: 通道连续 → BN 统计量计算快、ncnn Conv 优化好
  缺点: 逐像素操作需要跨大步跳跃 → cache miss 多

NHWC (TensorFlow/TFLite 默认): [N][H][W][C]
  内存: N₀H₀W₀C₀, N₀H₀W₀C₁, ..., N₀H₀W₀C_n, N₀H₀W₁C₀, ...
  优点: 一个像素的所有通道紧挨着 → 逐像素操作 (ReLU/Add) 很爽
  缺点: 通道间不连续 → Conv 的向量化难度大
```

> 💡 **直觉记忆**：NCHW 是"按颜色分堆"（所有红色放一起、所有蓝色放一起）。NHWC 是"按位置分堆"（左上角的所有颜色放一起、右下角的所有颜色放一起）。

## 3. 注意力机制——"全局依赖的建立者"

### 3.1 SDPA：五个步骤的精密舞蹈

```
Attention(Q, K, V) = softmax(Q·K^T / √d_k) · V

Step 1: Q·K^T    → "我的问题和你相关吗？" (n×n 的相似度矩阵)
Step 2: / √d_k    → "别让数值太大" (d=128 → scale≈0.088)
Step 3: + mask    → "你不能看到未来的内容" (因果掩码: 未来位置=−∞)
Step 4: softmax   → 变成概率分布 (每行加起来=1)
Step 5: · V       → 加权提取信息

Q: 我要找什么
K: 我有什么标签
V: 我有什么内容
→ Q 和 K 做匹配，匹配结果用来加权 V
```

### 3.2 GQA——"共享经济"在注意力中的应用

```
MHA: 每个 Q head 配一个独立的 KV head → 16 份 KV
GQA: 多个 Q head 共享一组 KV head     → 8 份 KV

ncnn 的实现:
  K 投影出来后只有 8 个头: [128, 8, seq]
  → ExpandDims: [128, 8, seq, 1]
  → Tile(2): [128, 8, seq, 2]
  → Reshape: [128, seq, 16]
  8 → 16，复制完成！省了一半 KV Cache
  
为什么 8 个 KV 就够给 16 个 Q 用？
  → Q head 之间本来就在关注差不多的东西
  → 让它们"拼车"几乎不损失质量
```

### 3.3 RoPE——不需要"记住"位置，只需要"感受"距离

```
RoPE 的优雅之处在于: 它编码的是相对位置

Q_pos_i · K_pos_j 
  = (旋转了 i 次的 q) · (旋转了 j 次的 k)
  = q · (旋转了 (j-i) 次的 k)
  ↑ 只依赖差值 (j-i)！不依赖 i 或 j 的绝对值！

这意味着:
  → 模型不需要学"位置 537 的特征是什么"
  → 只需要学"距离为 3 的 token 之间应该怎么交互"
  → 这个规律可以外推到训练时没见过的长度
```
